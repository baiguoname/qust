ta.cdllongline

CDLLONGLINE 形态指标。

namespace: ta returns: Expr

用法

Python
col('open', 'high', 'low', 'close').ta.cdllongline()

如果这个算子返回 Expr，通常可以继续链式调用 .alias(...)、.over(...)、.rolling(...) 或放入 qs.select(...)。

参数

无显式参数；输入来自当前表达式、绑定对象或当前命名空间。

输入和输出

项目说明
输入表达式open, high, low, close
返回类型Expr
输出语义Expr 指标表达式

完整例子：打印输入和输出

每个算子页面都必须有这一段完整例子：先打印输入表达式和输出表达式，再构造输入数据、打印输入数据、执行算子、打印输出结果。

完整例子 / Python
import polars as pl
import qust as qs
from qust import col

print("输入表达式:")
print("col('open', 'high', 'low', 'close')")
print("输出表达式:")
print("col('open', 'high', 'low', 'close').ta.cdllongline()")

data = pl.DataFrame(
    {
        "open": [10.0, 10.2, 10.1],
        "high": [10.4, 10.5, 10.3],
        "low": [9.9, 10.0, 10.0],
        "close": [10.2, 10.1, 10.3],
    }
)

print("输入 DataFrame:")
print(data)
print("调用:")
print("col('open', 'high', 'low', 'close').ta.cdllongline()")

try:
    expr = col('open', 'high', 'low', 'close').ta.cdllongline()
    out = qs.select(expr).calc_data(data)
    print("输出 DataFrame:")
    print(out)
except NotImplementedError as err:
    print("输出异常:")
    print(type(err).__name__ + ": " + str(err))

使用场景

用于技术指标和行情特征计算，常见输入是 close 或 open/high/low/close/volume。

当你需要 ta.cdllongline 这种语义时，优先使用这个算子，而不是在 Python 里提前把数据取出来手写循环；这样才能保留流式状态、参数联动、Wasm 路径和 Monitor 重算能力。

为什么这样设计

Qust 把这一类能力做成表达式节点，是为了把“用户想算什么”保留下来。只要语义还在表达式树里，后续就可以统一处理分组上下文、窗口状态、参数联动、Wasm 执行和 Monitor 可视化。

实际收益是：代码更短，但语义更完整；后续计算、画图、调参、保存和回调都能复用同一棵表达式树。

注意事项

状态：这个 API 当前会抛出 NotImplementedError；页面示例展示的是调用后的异常形态，而不是可计算指标输出。
  • 确认输入列顺序和列类型符合页面说明。
  • 需要分组时，把 over(...) 放在正确层级。
  • 需要 monitor 参数联动时，不要提前把 Params 转成普通 Python 数字。
