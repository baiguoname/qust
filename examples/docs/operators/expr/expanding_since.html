expr.expanding_since

上下文算子，条件重置版累积计算（expanding with reset）📌

namespace: expr returns: Expr

Python
qs.with_cols(
     (pl.col("datetime").dt.date() != pl.col("datetime").dt.date().shift(1)).fill_null(False).alias("is_new_day"),

 )

使用场景

用于构造表达式链路中的核心计算、上下文切换、调试或元数据操作。

当你需要 expr.expanding_since 这种语义时，优先使用这个算子，而不是在 Python 里提前把数据取出来手写循环；这样才能保留流式状态、参数联动、Wasm 路径和 Monitor 重算能力。

为什么这样设计

Qust 把这一类能力做成表达式节点，是为了把“用户想算什么”保留下来。只要语义还在表达式树里，后续就可以统一处理分组上下文、窗口状态、参数联动、Wasm 执行和 Monitor 可视化。

实际收益是：代码更短，但语义更完整；后续计算、画图、调参、保存和回调都能复用同一棵表达式树。

注意事项

- `reset_by` 不是布尔列：会报输入类型错误
- `reset_by` 返回多列：会报“必须单列”错误
- 重置信号持续为 `True`：每行都会重置，结果退化为“仅当前行值”
- 误以为重置从“下一行”生效：本算子是“当前行先重置再更新”
- 和 `over` 混用时分组条件不对：会导致不同标的状态串扰

**上下文算子**，条件重置版累积计算（`expanding` with reset）📌

作用 ✅
----
在标准 `expanding()` 语义上增加“重置触发条件”：
每当 `reset_by` 在某一行是 `True`，就先清空累积状态，再把当前行作为新的累积起点。

输入 📥
----
`reset_by: ExprInput`
- 语义：重置条件
- 类型要求：必须是**单列** `bool` 表达式（可空）
- 空值处理：`null` 会按 `False` 处理（即该行不触发重置）

逻辑 🔄
------
对每一行按顺序执行：

1. 先计算 `reset_by`
2. 若该行 `reset_by=True`，先清空内部状态
3. 再用当前行输入更新累积算子状态
4. 输出该行结果

这意味着“触发重置的这一行”会成为新窗口的第一行，而不是从下一行开始。

返回 📤
----
返回值仍是 `Expr`，其输出类型由内部累积算子决定，例如：
- `col("a").sum(axis=0).expanding_since(...)` -> 数值列（通常 `f64`/整型）
- `col("a").max(axis=0).expanding_since(...)` -> 与输入兼容的数值列
- `col("a").mean(axis=0).expanding_since(...)` -> `f64` 列

与其它上下文的关系 🧠
-------------------
- `expanding()`：从序列起点一直累积，不会中途清零
- `expanding_since(reset_by)`：遇到 `reset_by=True` 时中途清零后继续累积
- `rolling(...)`：固定窗口，依赖 `retract`，语义与本算子不同

典型用途 🎯
--------
- 按交易日重置日内累积指标（如日内累积成交量、日内高低点统计）
- 按“开仓信号”重置开仓后状态（如开仓后累计收益、开仓后最大值追踪）
- 按事件边界重启统计（如 regime 切换、风控状态切换）

示例 💡
----
1. 每逢新交易日重置，计算日内累计成交量：

>>> qs.with_cols(
...     (pl.col("datetime").dt.date() != pl.col("datetime").dt.date().shift(1)).fill_null(False).alias("is_new_day"),
...     col("volume").sum(axis=0).expanding_since("is_new_day").alias("volume_cum_day")
... )

2. 开仓信号触发后重置，计算“开仓以来最高价”：

>>> qs.with_cols(
...     col("high").max(axis=0).expanding_since("open_long_sig").alias("high_since_entry")
... )

3. 与 `over` 组合：按品种各自独立重置并累积：

>>> qs.with_cols(
...     col("pnl").sum(axis=0).expanding_since("reset_sig").over("symbol").alias("pnl_cum_by_symbol")
... )

可能出错的地方 ⚠️
--------------
- `reset_by` 不是布尔列：会报输入类型错误
- `reset_by` 返回多列：会报“必须单列”错误
- 重置信号持续为 `True`：每行都会重置，结果退化为“仅当前行值”
- 误以为重置从“下一行”生效：本算子是“当前行先重置再更新”
- 和 `over` 混用时分组条件不对：会导致不同标的状态串扰
