expr.with_cols

无状态算子， 和select类似，只不过保留输入列，如果新生成列和输入列有重复列，会只保留新生成的重复列

namespace: expr returns: Expr

用法

Python
col("a", "b").with_cols(
     (col("a") + col("b")).alias("sum_ab"),
     (col("a") * 2).alias("a"),
 )

如果这个算子返回 Expr，通常可以继续链式调用 .alias(...)、.over(...)、.rolling(...) 或放入 qs.select(...)。

参数

参数类型默认值说明
*argsExprInput可选可变位置参数

输入和输出

项目说明
输入表达式当前表达式，例如 col("x")；具体列宽由底层算子校验。
返回类型Expr
输出语义返回新的表达式或 API 调用结果；列名通常由输入列名、alias 或底层算子决定。

完整例子：打印输入和输出

每个算子页面都必须有这一段完整例子：先打印输入表达式和输出表达式，再构造输入数据、打印输入数据、执行算子、打印输出结果。

完整例子 / Python
import polars as pl
import qust as qs
from qust import col

print("输入表达式:")
print('col("a", "b")')
print("输出表达式:")
print('col("a", "b").with_cols(\n     (col("a") + col("b")).alias("sum_ab"),\n     (col("a") * 2).alias("a"),\n )')

data = pl.DataFrame(
    {
        "a": [3, 1, 2],
        "b": [2, 3, 4],
    }
)

print("输入 DataFrame:")
print(data)

expr = col("a", "b").with_cols(
     (col("a") + col("b")).alias("sum_ab"),
     (col("a") * 2).alias("a"),
 )
df = qs.select(expr)
out = df.calc_data(data)

print("输出 DataFrame:")
print(out)

使用场景

用于构造表达式链路中的核心计算、上下文切换、调试或元数据操作。

当你需要 expr.with_cols 这种语义时，优先使用这个算子，而不是在 Python 里提前把数据取出来手写循环；这样才能保留流式状态、参数联动、Wasm 路径和 Monitor 重算能力。

为什么这样设计

Qust 把这一类能力做成表达式节点，是为了把“用户想算什么”保留下来。只要语义还在表达式树里，后续就可以统一处理分组上下文、窗口状态、参数联动、Wasm 执行和 Monitor 可视化。

实际收益是：代码更短，但语义更完整；后续计算、画图、调参、保存和回调都能复用同一棵表达式树。

注意事项

  • 确认输入列顺序和列类型符合页面说明。
  • 需要分组时，把 over(...) 放在正确层级。
  • 需要 monitor 参数联动时，不要提前把 Params 转成普通 Python 数字。
