expr.get_by_index

重排行算子，按参数表达式给出的 UInt32 行号，从当前输入里取行。

namespace: expr returns: Expr

用法

Python
col("price").get_by_index(
     col("open_sig", "exit_sig").stra.trade_row()
 )

如果这个算子返回 Expr，通常可以继续链式调用 .alias(...)、.over(...)、.rolling(...) 或放入 qs.select(...)。

参数

参数类型默认值说明
indexExprInput必填位置参数
keepboolFalse位置参数

输入和输出

项目说明
输入表达式当前表达式，例如 col("x")；具体列宽由底层算子校验。
返回类型Expr
输出语义返回新的表达式或 API 调用结果；列名通常由输入列名、alias 或底层算子决定。

使用场景

用于构造表达式链路中的核心计算、上下文切换、调试或元数据操作。

当你需要 expr.get_by_index 这种语义时，优先使用这个算子，而不是在 Python 里提前把数据取出来手写循环；这样才能保留流式状态、参数联动、Wasm 路径和 Monitor 重算能力。

为什么这样设计

Qust 把这一类能力做成表达式节点，是为了把“用户想算什么”保留下来。只要语义还在表达式树里，后续就可以统一处理分组上下文、窗口状态、参数联动、Wasm 执行和 Monitor 可视化。

实际收益是：代码更短，但语义更完整；后续计算、画图、调参、保存和回调都能复用同一棵表达式树。

注意事项

- `index` 没有以前缀 `UInt32` 列开头：会报输入类型错误
- `index` 含 `null`：会直接报错
- 普通上下文下，`index` 必须是整条输入累计行号
- 外层 `over(...)` 上下文下，`index` 必须是同一 group 内的组内行号
- `index` 子表达式自己带 `over(...)` 会报错；请把 `over(...)` 写在外层

**重排行算子**，按参数表达式给出的 `UInt32` 行号，从当前输入里取行。

核心语义 ✅
----------
- 当前输入可以是 1 列或多列。
- `index` 必须是一个表达式，计算结果必须以 **一列或多列 `UInt32`** 开头。
- 开头连续的 `UInt32` 列会被当作取数 index。
- 如果后面还有非 `UInt32` 列，这些列是 payload，不参与取数。
- 输出按“输入列 × index 列”做笛卡尔展开，只使用前缀 `UInt32` index 列。
- 输出列名规则：
  `输入列名_at_索引列名`

例如：
- 输入列：`A, B`
- `index` 输出列：`a, b, c`
- 结果列：
  `A_at_a, A_at_b, A_at_c, B_at_a, B_at_b, B_at_c`

`keep=True` 时，会先把 `index` 自己的输出列一并保留，再接结果列。
这会保留 `trade_row()` 双边交易输出里的 `side: u8`：
- `a, b, c, A_at_a, A_at_b, A_at_c, B_at_a, B_at_b, B_at_c`

输入 📥
-------
- 函数签名：
  `get_by_index(index: ExprInput, keep: bool = False)`
- `index`：
  - 必须以前缀 `UInt32` 列开头
  - 前缀 `UInt32` 列会被当作 index
  - 后缀非 `UInt32` 列会被当作 payload
  - 不允许出现 `null`
  - 行顺序按 `index` 原顺序保留
  - 重复 index 允许，结果会重复取对应行
- `keep`：
  - `False`：只返回取出的结果列
  - `True`：先返回 `index` 的全部输出列（包括 payload），再返回取出的结果列

返回 📤
-------
- 返回类型：`Expr`
- 返回语义：返回一个新的表达式；上下文直接穿透，不主动改写 `over/batch` 等上下文。
- 在普通上下文下，`index` 按整条输入累计的行号解释。
- 在外层 `over(...)` 上下文下，`index` 按同一个 group 内从 0 开始的组内行号解释。
- `index` 自己不能再带 `over(...)`；需要把 `over(...)` 放在 `get_by_index(...)` 外层，让输入和 index 继承同一个上下文。
