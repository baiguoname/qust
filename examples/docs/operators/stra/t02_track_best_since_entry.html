使用场景

用于策略信号、持仓状态、出入场和交易行号生成。

当你需要 stra.t02_track_best_since_entry 这种语义时，优先使用这个算子，而不是在 Python 里提前把数据取出来手写循环；这样才能保留流式状态、参数联动、Wasm 路径和 Monitor 重算能力。

为什么这样设计

策略和回测算子被设计成表达式，是为了让信号、持仓、成交、收益可以和 over/rolling/monitor 组合。这比在 Python for 循环里维护状态更容易复用，也能在分组和流式数据里保持一致语义。

实际收益是：策略信号、交易轮次、持仓收益和交易明细可以直接接到图表和 DataPool，方便定位每一笔交易。

注意事项

- 输入列顺序错误：该算子按固定位置读取 4 列，不按列名匹配
- `entry_price` 不是“持仓状态入场价轨迹”：会造成极值初始化错误
- `favorable_1` 方向传反：
  - 多头误传 `low_1`
  - 空头误传 `high_1`
- `open_sig` 连续为 `True`：会每行重置极值，无法得到正常“持仓内累积极值”
- 忘记接 `expanding()`：结果不是完整时序状态列

**行算子**，跟踪“入场后有利方向的极值状态”📌

作用 ✅
----
该算子用于在持仓期间维护一个“最有利价格基准”：
- 多头语义下维护 `high_since_entry`
- 空头语义下维护 `low_since_entry`

它常作为后续回撤止盈/移动止损的基础状态输入。

输入 📥
----
输入是 `col(open_sig, exit_sig, favorable_1, entry_price)`，顺序固定为 4 列：

1. `open_sig`: `bool | None`
2. `exit_sig`: `bool | None`
3. `favorable_1`: `f64 | None`
4. `entry_price`: `f64 | None`

说明：
- `open_sig` / `exit_sig` 的空值按 `False` 处理
- `favorable_1`：
  - `reverse=False`（多头）通常传 `high_1`
  - `reverse=True`（空头）通常传 `low_1`
- `entry_price` 通常来自 `t02_track_entry_price` 的输出

参数 ⚙️
----
`reverse: bool = False`
- `False`：多头语义（跟踪最大值）
- `True`：空头语义（跟踪最小值）

状态逻辑 🔄
--------
每行更新顺序如下（先平，再开，再续）：

1. 若 `exit_sig=True`：清空内部极值状态 `best`
2. 若 `open_sig=True`：`best = entry_price`
3. 否则（非开仓行）：
   - 若 `best` 与 `favorable_1` 都非空：
     - 多头：`best = max(best, favorable_1)`
     - 空头：`best = min(best, favorable_1)`
4. 输出当前 `best`

同一行 `exit_sig=True` 且 `open_sig=True` 时，结果是：
- 先清旧仓状态
- 再用本行 `entry_price` 初始化新仓极值

返回 📤
----
返回 1 列，类型 `f64`（可空）：
- `reverse=False`：列名语义为 `high_since_entry_expr`
- `reverse=True`：列名语义为 `low_since_entry_expr`
- 非持仓或初始化失败时输出 `null`

上下文与计算语义 🧠
--------------
- 这是**行算子**
- 行算子若不接上下文算子（如 `expanding` / `rolling_*` / `group_by`），
  不具备“整列状态序列”语义
- 典型用法：
  `col(...).stra.t02_track_best_since_entry(...).expanding()`

典型用途 🎯
--------
- 与当前价比较构造回撤触发：
  - 多头：`close / high_since_entry < thre`
  - 空头：`close / low_since_entry > 1 / thre`
- 构造“开仓以来最好价”序列用于调试和可视化

示例 💡
----
1. 多头：跟踪开仓后最高有利价（`high_1`）：

>>> qs.with_cols(
...     col("open_sig", "exit_sig", "entry_now")
...         .stra
...         .t02_track_entry_price()
...         .expanding()
...         .alias("entry_price"),
...     col("open_sig", "exit_sig", "high_1", "entry_price")
...         .stra
...         .t02_track_best_since_entry(False)
...         .expanding()
...         .alias("high_since_entry")
... )

2. 空头：跟踪开仓后最低有利价（`low_1`）：

>>> qs.with_cols(
...     col("open_sig", "exit_sig", "entry_now")
...         .stra
...         .t02_track_entry_price()
...         .expanding()
...         .alias("entry_price"),
...     col("open_sig", "exit_sig", "low_1", "entry_price")
...         .stra
...         .t02_track_best_since_entry(True)
...         .expanding()
...         .alias("low_since_entry")
... )

可能出错的地方 ⚠️
--------------
- 输入列顺序错误：该算子按固定位置读取 4 列，不按列名匹配
- `entry_price` 不是“持仓状态入场价轨迹”：会造成极值初始化错误
- `favorable_1` 方向传反：
  - 多头误传 `low_1`
  - 空头误传 `high_1`
- `open_sig` 连续为 `True`：会每行重置极值，无法得到正常“持仓内累积极值”
- 忘记接 `expanding()`：结果不是完整时序状态列
