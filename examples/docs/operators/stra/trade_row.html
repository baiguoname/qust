stra.trade_row

交易轮次算子，把开平仓信号转换成完整成交 round 的行号表。

namespace: stra returns: Expr

用法

Python
col("open_sig", "exit_sig").stra.trade_row()

如果这个算子返回 Expr，通常可以继续链式调用 .alias(...)、.over(...)、.rolling(...) 或放入 qs.select(...)。

参数

无显式参数；输入来自当前表达式、绑定对象或当前命名空间。

输入和输出

项目说明
输入表达式1. 2 列： - open - exit
返回类型Expr
输出语义- entry_index - exit_index - index_group_id 2. 4 列： - open_long - exit_long - open_short - exit_short

使用场景

用于策略信号、持仓状态、出入场和交易行号生成。

当你需要 stra.trade_row 这种语义时，优先使用这个算子，而不是在 Python 里提前把数据取出来手写循环；这样才能保留流式状态、参数联动、Wasm 路径和 Monitor 重算能力。

为什么这样设计

策略和回测算子被设计成表达式，是为了让信号、持仓、成交、收益可以和 over/rolling/monitor 组合。这比在 Python for 循环里维护状态更容易复用，也能在分组和流式数据里保持一致语义。

实际收益是：策略信号、交易轮次、持仓收益和交易明细可以直接接到图表和 DataPool，方便定位每一笔交易。

注意事项

  • 确认输入列顺序和列类型符合页面说明。
  • 需要分组时，把 over(...) 放在正确层级。
  • 需要 monitor 参数联动时，不要提前把 Params 转成普通 Python 数字。

**交易轮次算子**，把开平仓信号转换成完整成交 round 的行号表。

输入形态 ✅
----------
1. 2 列：
   - `open`
   - `exit`
   输出：
   - `entry_index`
   - `exit_index`
   - `__index_group_id`

2. 4 列：
   - `open_long`
   - `exit_long`
   - `open_short`
   - `exit_short`
   输出：
   - `entry_index`
   - `exit_index`
   - `side`（`u8`，`1` 表示 long，`2` 表示 short）
   - `__index_group_id`

说明 🧠
-------
- 内部会自动补一列行号 index
- 未平仓的最后一笔不会输出，所以输出行数可以少于输入行数
- 和 `over(...)` 组合时，会按 group 分开维护持仓状态
- 普通上下文下，输出 index 是整条输入累计行号，`__index_group_id` 固定为 `0`
- 外层 `over(...)` 上下文下，输出 index 是每个 group 内从 0 开始的组内行号，`__index_group_id` 是对应 over group id

返回 📤
-------
- 返回类型：`Expr`
- 返回语义：返回交易轮次表，可继续接 `get_by_index(...)` 等算子
