dataframe.save

将当前 DataFrame 表达式与配置序列化保存到磁盘。

namespace: dataframe returns: 未显式标注

用法

Python
df.save("/tmp/qust_df.bin")

如果这个算子返回 Expr，通常可以继续链式调用 .alias(...)、.over(...)、.rolling(...) 或放入 qs.select(...)。

完整例子：打印输入和输出

每个算子页面都必须有这一段完整例子：先打印输入表达式和输出表达式，再构造输入数据、打印输入数据、执行算子、打印输出结果。

完整例子 / Python
import polars as pl
import qust as qs
from qust import col
from qust.dataframe import DataFrame

print("输入表达式:")
print('df')
print("输出表达式:")
print('df.save("/tmp/qust_df.bin")')

data = pl.DataFrame({"x": [1.0, 2.0, 3.0]})
path = "/tmp/qust_df.bin"
df = qs.select(col("x").mean())

print("输入 DataFrame:")
print(data)
df.save(path)
loaded = DataFrame.load(path)
out = loaded.calc_data(data)

print("输出 DataFrame:")
print(out)

使用场景

用于构造、执行、保存和恢复 Qust DataFrame 执行计划。

当你需要 dataframe.save 这种语义时，优先使用这个算子，而不是在 Python 里提前把数据取出来手写循环；这样才能保留流式状态、参数联动、Wasm 路径和 Monitor 重算能力。

为什么这样设计

Qust DataFrame 是执行计划，不是已经计算完的数据。这样同一个计划可以反复喂不同数据、参数或 monitor session，适合策略研究和浏览器交互。

实际收益是：代码更短，但语义更完整；后续计算、画图、调参、保存和回调都能复用同一棵表达式树。

注意事项

  • 确认输入列顺序和列类型符合页面说明。
  • 需要分组时，把 over(...) 放在正确层级。
  • 需要 monitor 参数联动时，不要提前把 Params 转成普通 Python 数字。
