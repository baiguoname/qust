batch_stock.bt_intraday

股票日内换仓回测（先卖后买，含 T+1 与涨跌停/异常约束）。

namespace: batch_stock returns: Expr

用法

Python
col('datetime', 'code', 'w', 'price', 'flag', 'limit_up', 'limit_down').batch_stock.bt_intraday(100.0, 0.0003)

如果这个算子返回 Expr，通常可以继续链式调用 .alias(...)、.over(...)、.rolling(...) 或放入 qs.select(...)。

参数

参数类型默认值说明
init_moneyfloat100.0位置参数
fee_ratefloat | Params0.0003位置参数

参数补充说明

init_money:
    初始资金（仅第一期使用）。
fee_rate:
    手续费率，支持 `float` 或 `Params`。

输入和输出

项目说明
输入表达式1. datetime: datetime[ms]，当前输入块内必须唯一 2. code: str，全市场股票代码 3. w: f64，目标权重（通常和约等于 1） 4. price: f64，当前成交价格 5. flag: bool，是否可交易（True=正常） 6. limit_up: f64，涨停上限 7. limit_down: f64，跌停下限
返回类型Expr
输出语义hold_num, hold_money, trade_money, fee, idle_money, total_assets

使用场景

用于股票批处理/批内回测类计算。

当你需要 batch_stock.bt_intraday 这种语义时，优先使用这个算子，而不是在 Python 里提前把数据取出来手写循环；这样才能保留流式状态、参数联动、Wasm 路径和 Monitor 重算能力。

为什么这样设计

批内/截面算子把“按日期、代码或批次计算”的边界留在表达式里。这样排序、rank、lead/lag、回归等操作可以准确接受上下文，而不是提前把 DataFrame 切碎。

实际收益是：截面和批内计算不会丢掉分组边界，跨股票/合约的分析更容易写对。

注意事项

- 先卖后买。
- 若 `flag=False` 或 `price > limit_up` 或 `price < limit_down`，该股票当期禁止买卖。
- T+1：当天买入仓位当天不可卖；卖出按“先卖旧仓”执行。
- 若次日股票缺失（退市/停牌缺行），内部按上次价格补价并按目标仓位处理。
