expr.rolling_dynamic

上下文，变化窗口的滚动计算

namespace: expr returns: Expr

用法

Python
col("x").rolling_dynamic(window_size, 1000)

如果这个算子返回 Expr，通常可以继续链式调用 .alias(...)、.over(...)、.rolling(...) 或放入 qs.select(...)。

参数

参数类型默认值说明
window_sizeExprInput必填位置参数
window_maxint1000位置参数
min_samplesint | Params | NoneNone位置参数

参数补充说明

`window_size`: `str` | `pl.Expr` | `Expr`, 选择窗口列, 这列需要能被cast成整数，非空, 大于0

`window_max`: 最大可能的滚动窗口，设置过大可能会影响内存

`min_samples`: 最小有效值个数

输入和输出

项目说明
输入表达式当前表达式，例如 col("x")；具体列宽由底层算子校验。
返回类型Expr
输出语义- 返回类型：Expr - 上面的 DataFrame 输入 / 调用 / 输出 已给出真实打印结果。

import qust as qs
from qust import col

print("输入表达式:")
print('col("x")')

print('col("x").rolling_dynamic(window_size, 1000)')

data = pl.DataFrame(
    {
        "x": [1.0, 2.0, 3.0],
    }

expr = col("x").rolling_dynamic(window_size, 1000)
df = qs.select(expr)
out = df.calc_data(data)

print("输出 DataFrame:")
print(out)

使用场景

用于构造表达式链路中的核心计算、上下文切换、调试或元数据操作。

当你需要 expr.rolling_dynamic 这种语义时，优先使用这个算子，而不是在 Python 里提前把数据取出来手写循环；这样才能保留流式状态、参数联动、Wasm 路径和 Monitor 重算能力。

为什么这样设计

Qust 把这一类能力做成表达式节点，是为了把“用户想算什么”保留下来。只要语义还在表达式树里，后续就可以统一处理分组上下文、窗口状态、参数联动、Wasm 执行和 Monitor 可视化。

实际收益是：代码更短，但语义更完整；后续计算、画图、调参、保存和回调都能复用同一棵表达式树。

注意事项

  • 确认输入列顺序和列类型符合页面说明。
  • 需要分组时，把 over(...) 放在正确层级。
  • 需要 monitor 参数联动时，不要提前把 Params 转成普通 Python 数字。
