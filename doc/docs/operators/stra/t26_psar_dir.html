stra.t26_psar_dir

行算子，按 T26 源逻辑维护 PSAR 方向状态列 🚀

namespace: stra returns: Expr

用法

Python
col("high", "low", "close").stra.t26_psar_dir(0.02, 0.2).expanding().alias("sar_dir")

如果这个算子返回 Expr，通常可以继续链式调用 .alias(...)、.over(...)、.rolling(...) 或放入 qs.select(...)。

参数

参数类型默认值说明
af_stepfloat | Params必填位置参数
af_limitfloat | Params必填位置参数

参数补充说明

- `af_step`: PSAR 加速步长
- `af_limit`: PSAR 加速上限

输入和输出

项目说明
输入表达式- -1.0：对应多头方向信号 - 1.0：对应空头方向信号 该实现保留 T26 原逻辑的关键细节： - 初始化方向判断（h >= prev_high） - prev_low/prev2_low 与 prev_high/prev2_high 的夹逼修正 - 极值刷新时 af 递增并受 af_limit 限制
返回类型Expr
输出语义- 返回 1 列：sar_dir（f64） 使用场景 🎯 - 与 t26_adx 组合做 ADX 区间过滤后的反转入场

使用场景

用于策略信号、持仓状态、出入场和交易行号生成。

当你需要 stra.t26_psar_dir 这种语义时，优先使用这个算子，而不是在 Python 里提前把数据取出来手写循环；这样才能保留流式状态、参数联动、Wasm 路径和 Monitor 重算能力。

为什么这样设计

策略和回测算子被设计成表达式，是为了让信号、持仓、成交、收益可以和 over/rolling/monitor 组合。这比在 Python for 循环里维护状态更容易复用，也能在分组和流式数据里保持一致语义。

实际收益是：策略信号、交易轮次、持仓收益和交易明细可以直接接到图表和 DataPool，方便定位每一笔交易。

注意事项

  • 确认输入列顺序和列类型符合页面说明。
  • 需要分组时，把 over(...) 放在正确层级。
  • 需要 monitor 参数联动时，不要提前把 Params 转成普通 Python 数字。

**行算子**，按 T26 源逻辑维护 PSAR 方向状态列 🚀

作用 ✅
----
输入 `col(high, low, close)`，输出 1 列 `sar_dir`（`f64`）：
- `-1.0`：对应多头方向信号
- `1.0`：对应空头方向信号

该实现保留 T26 原逻辑的关键细节：
- 初始化方向判断（`h >= prev_high`）
- `prev_low/prev2_low` 与 `prev_high/prev2_high` 的夹逼修正
- 极值刷新时 `af` 递增并受 `af_limit` 限制

输入 📥
----
输入顺序固定 3 列：
1. `high`: `f64 | None`
2. `low`: `f64 | None`
3. `close`: `f64 | None`

参数 ⚙️
----
- `af_step`: PSAR 加速步长
- `af_limit`: PSAR 加速上限

返回 📤
----
- 返回 1 列：`sar_dir`（`f64`）

使用场景 🎯
----
- 与 `t26_adx` 组合做 ADX 区间过滤后的反转入场

示例 💡
----
>>> col("high", "low", "close").stra.t26_psar_dir(0.02, 0.2).expanding().alias("sar_dir")
