使用场景

用于策略信号、持仓状态、出入场和交易行号生成。

当你需要 stra.t06_ama_band 这种语义时，优先使用这个算子，而不是在 Python 里提前把数据取出来手写循环；这样才能保留流式状态、参数联动、Wasm 路径和 Monitor 重算能力。

为什么这样设计

策略和回测算子被设计成表达式，是为了让信号、持仓、成交、收益可以和 over/rolling/monitor 组合。这比在 Python for 循环里维护状态更容易复用，也能在分组和流式数据里保持一致语义。

实际收益是：策略信号、交易轮次、持仓收益和交易明细可以直接接到图表和 DataPool，方便定位每一笔交易。

注意事项

- 输入列顺序错误：该算子按固定位置读取第 1 列
- 忘记接 `expanding()`：拿不到完整时序状态列
- 参数过小/过极端会导致信号抖动明显增大

**行算子**，按 T06 源逻辑计算 `ama` 与 `ifilter` 双状态列 🚀

作用 ✅
----
输入一列 `close`，输出两列：
1. `ama`: 基于效率比平滑后的双层均值状态
2. `ifilter`: 由 `ama` 差分波动 + 低效率补偿组成的带宽

该算子已下沉到 Rust 行执行，适合长序列回测中的高频状态更新。

输入 📥
----
输入是 `col(close)`，顺序固定 1 列：
1. `close`: `f64 | None`

参数 ⚙️
----
- `period`: 效率比主周期（最小按 2 处理）
- `period1`: 噪声/信号回看周期（最小按 2 处理）
- `filt`: 波动带宽系数
- `filter_period`: `ama` 差分标准差窗口（最小按 2 处理）
- `smooth_len`: 效率比平滑指数
- `eff`: 低效率阈值，`ef < eff` 时追加补偿
- `dema_len`: 双层均值窗口（最小按 2 处理）

逻辑 🔄
------
每行按顺序更新：
1. 基于历史 `close` 维护差分噪声 `diff_hist`
2. 计算 `ef = signal / max(noise, 1e-7)`
3. 得到平滑系数并更新 `ema`
4. 对 `ema` 做双层均值得到 `ama`
5. 计算 `ama` 差分滚动标准差 `st`
6. 输出：
   - `ama`
   - `ifilter = filt * st + (ef * 5.0 if ef < eff else 0.0)`

返回 📤
----
返回 2 列（`f64`）：
- `ama`
- `ifilter`

示例 💡
----
>>> col("close").stra.t06_ama_band(
...     period=15,
...     period1=5,
...     filt=7.0,
...     filter_period=37,
...     smooth_len=3.1,
...     eff=0.78,
...     dema_len=8,
... ).expanding().alias("ama", "ifilter")

适用场景 🎯
----
- T06 风格“均线拐点 + 带宽”信号
- 需要复用 `ama/ifilter` 中间状态的趋势策略

可能出错的地方 ⚠️
--------------
- 输入列顺序错误：该算子按固定位置读取第 1 列
- 忘记接 `expanding()`：拿不到完整时序状态列
- 参数过小/过极端会导致信号抖动明显增大
