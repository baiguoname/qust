stock.residual

批算子，无状态保留，计算线性回归的残差

namespace: stock returns: Expr

用法

Python
col("y", "x1", "x2").stock.residual()

如果这个算子返回 Expr，通常可以继续链式调用 .alias(...)、.over(...)、.rolling(...) 或放入 qs.select(...)。

import polars as pl
import qust as qs
from qust import col

print("输入表达式:")
print('col("y", "x1", "x2")')
print("输出表达式:")
print('col("y", "x1", "x2").stock.residual()')

data = pl.DataFrame(
    {
        "y": [2.0, 3.5, 3.0],
        "x1": [1.0, 2.0, 3.0],
        "x2": [2.0, 0.5, 1.5],
    }
)

print("输入 DataFrame:")
print(data)

expr = col("y", "x1", "x2").stock.residual()
df = qs.select(expr)
out = df.calc_data(data)

print("输出 DataFrame:")
print(out)

使用场景

用于股票统计建模相关表达式，例如 OLS 和残差。

当你需要 stock.residual 这种语义时，优先使用这个算子，而不是在 Python 里提前把数据取出来手写循环；这样才能保留流式状态、参数联动、Wasm 路径和 Monitor 重算能力。

为什么这样设计

批内/截面算子把“按日期、代码或批次计算”的边界留在表达式里。这样排序、rank、lead/lag、回归等操作可以准确接受上下文，而不是提前把 DataFrame 切碎。

实际收益是：截面和批内计算不会丢掉分组边界，跨股票/合约的分析更容易写对。

注意事项

- 参数类型与预期不一致会导致运行时报错或返回空值。
- 在上下文算子链中使用时，请确认调用顺序与分组/窗口语义一致。
- 若上游列名、列顺序或数据类型变化，结果可能与示例不一致。
