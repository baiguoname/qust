qs.pms

创建参数对象（统一入口）。

namespace: top-level returns: Params

用法

Python
qs.pms(2, 120).title("window").value(20).step(1)

如果这个算子返回 Expr，通常可以继续链式调用 .alias(...)、.over(...)、.rolling(...) 或放入 qs.select(...)。

使用场景

用于 Python 用户的顶层便捷入口。

当你需要 qs.pms 这种语义时，优先使用这个算子，而不是在 Python 里提前把数据取出来手写循环；这样才能保留流式状态、参数联动、Wasm 路径和 Monitor 重算能力。

为什么这样设计

Qust 把这一类能力做成表达式节点，是为了把“用户想算什么”保留下来。只要语义还在表达式树里，后续就可以统一处理分组上下文、窗口状态、参数联动、Wasm 执行和 Monitor 可视化。

实际收益是：代码更短，但语义更完整；后续计算、画图、调参、保存和回调都能复用同一棵表达式树。

注意事项

- 字符串枚举参数不支持 `.step(...)`。
- `pms` 只能接收两参，传入旧签名会抛 `TypeError`。

创建参数对象（统一入口）。

仅支持两个位置参数：
1. 数值参数：`pms(start, end)`
2. 字符串枚举参数：`pms(pool, None)`，其中 `pool` 是 `list[str]`

说明：
- `title/value/step` 不再作为 `pms` 位置参数传入。
- 统一通过链式方法设置：
  - `.title("name")`
  - `.value(v)`
  - `.step(v)`

示例：
- 整数参数：`pms(2, 120).title("n").value(20).step(2)`
- 浮点参数：`pms(0.1, 5.0).title("k").value(0.7).step(0.1)`
- 枚举参数：`pms(["a", "b", "c"], None).title("sig_name").value("b")`

注意：
- 字符串枚举参数不支持 `.step(...)`。
- `pms` 只能接收两参，传入旧签名会抛 `TypeError`。
