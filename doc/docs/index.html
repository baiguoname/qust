Python API: qust streaming execution core Monitor protocol Arrow packet data plane Wasm / Pyodide

普通使用者

浏览器和分享用户

导入方式

Python
import polars as pl
import qust as qs
from qust import col, pms, Monitor, x_slider, scatter_select
对象 用途 最常见写法
qs.DataFrame 创建 Polars DataFrame。注意它返回的是 polars.DataFrame，不是执行计划。 qs.DataFrame({"a": [1, 2]})
qs.select 构造只输出指定表达式的执行计划。 qs.select(col("close"))
qs.with_cols 构造在原数据上追加/覆盖列的执行计划。 qs.with_cols((col("a") + 1).alias("a1"))
col 表达式入口，选择列、常量、Polars 表达式或 Qust 表达式。 col("open", "close")
pms 创建 live 参数，让 monitor 参数面板能驱动重算。 pms(2, 120).title("window").value(20)
不要把 qs.DataFrame(...) 和执行计划混为一谈。 qs.DataFrame({...}) 是数据；qs.select(...)、qs.with_cols(...) 返回的是可以对数据执行的计划对象。

十分钟跑通

这一节用一份很小的行情数据演示完整流程：准备数据、构造表达式、执行计算、查看输出、画 monitor。

准备输入数据

Python
import polars as pl
import qust as qs
from qust import col

data = pl.DataFrame(
    {
        "datetime": [
            "2026-01-01 09:30:00",
            "2026-01-01 09:31:00",
            "2026-01-01 09:32:00",
            "2026-01-01 09:30:00",
            "2026-01-01 09:31:00",
            "2026-01-01 09:32:00",
        ],
        "code": ["AAA", "AAA", "AAA", "BBB", "BBB", "BBB"],
        "open": [10.0, 10.2, 10.1, 20.0, 20.5, 20.3],
        "high": [10.4, 10.5, 10.3, 20.7, 20.8, 20.4],
        "low": [9.9, 10.0, 10.0, 19.8, 20.1, 20.0],
        "close": [10.2, 10.1, 10.3, 20.5, 20.3, 20.4],
        "volume": [1000, 900, 1200, 1800, 1600, 1700],
    }
).with_columns(pl.col("datetime").str.to_datetime())

print(data)

每一行是一根 K 线或一个时间点的行情。

输出

第一个表达式执行计划

qs.select(...) 会创建一个执行计划。只有调用 calc_data(data) 时，表达式才会真正对输入数据执行。

Python
df = qs.select(
    col("datetime"),
    col("code"),
    (col("close") - col("open")).alias("body"),
    (col("high") - col("low")).alias("range"),
)

out = df.calc_data(data)
print(out)

输入

data: Polars DataFrame
需要至少包含 datetime, code, open, high, low, close。

输出

shape: (6, 4)
┌─────────────────────┬──────┬──────┬───────┐
│ datetime            ┆ code ┆ body ┆ range │
│ ---                 ┆ ---  ┆ ---  ┆ ---   │
│ datetime[μs]        ┆ str  ┆ f64  ┆ f64   │
╞═════════════════════╪══════╪══════╪═══════╡
│ 2026-01-01 09:30:00 ┆ AAA  ┆ 0.2  ┆ 0.5   │
│ 2026-01-01 09:31:00 ┆ AAA  ┆ -0.1 ┆ 0.5   │
│ ...                 ┆ ...  ┆ ...  ┆ ...   │
└─────────────────────┴──────┴──────┴───────┘

加一个分组滚动均线

金融数据通常有多个标的。时序计算一般要按 code 分组，否则 AAA 的上一行可能串到 BBB。

Python
df = qs.select(
    col("datetime"),
    col("code"),
    col("close"),
    col("close").mean().rolling(2, 1).over("code").alias("ma2"),
)

print(df.calc_data(data))
新手口诀：只要是“每个标的自己看自己的过去”，表达式最后通常要加 .over("code")。 只要是“同一个时刻所有标的一起排名”，通常是 .over("datetime")。

数据模型

Otters / Qust 的 Python 体验接近 Polars 表达式系统，但执行模型更偏流式状态计算。

Polars DataFrame 是输入数据。 你用 pl.DataFrame、qs.DataFrame 或 load_data 得到数据。
Qust DataFrame 是执行计划。 你用 qs.select、qs.with_cols、DataFrame().select 组合表达式。
Expr 是抽象计算。 col("close").shift(1) 不会马上算，只有进入 calc_data 或 monitor session 才执行。
Monitor 是展示和交互层。 Monitor 不替你决定计算语义，它显示表达式执行后生成的 plot packet，并通过协议发参数/查询命令。

执行计划和输入数据的关系

Python
plan = qs.select((col("close") / col("open") - 1).alias("ret"))

chunk_1 = data.head(3)
chunk_2 = data.tail(3)

out_1 = plan.calc_data(chunk_1)
out_2 = plan.calc_data(chunk_2)

如果计划里包含 rolling、expanding、shift、over 等状态算子，同一个 plan 对象会保留内部状态。也就是说，第二次 calc_data 不是从空状态开始。

需要重新开始时：重新构造一个新的执行计划，或者用 snapshot() 在某个状态点分叉。

表达式与上下文

表达式是 Qust 的核心。你可以把它理解为“描述如何从输入列计算输出列的一棵树”。 表达式本身不持有数据，执行计划把表达式应用到输入数据上。

最小表达式

Python
col("close")                      # 选择 close 列
col("open", "close")              # 选择两列
col.lit(1.0)                      # 字面量常量
(col("close") - col("open"))      # 表达式组合
(col("close") / col("open") - 1)  # 收益率

上下文是什么

同一个算子放在不同上下文里，语义会变。下面是最常见的三类上下文。

上下文 含义 例子
rolling(n) 只看当前行之前最近 n 行。 col("close").mean().rolling(20)
expanding() 从开始到当前行累计。 col("volume").sum().expanding()
over(...) 按列分组，每个组独立维护状态。 col("close").shift(1).over("code")
batch(...) 按标签维护批缓存，并向下游传播 update/retract。 col("alpha").batch("datetime")
snapshot_by(...) 按标签维护快照基线，标签回流时回放，保证状态一致。 col("x").mean().rolling(3).snapshot_by("datetime")

select / with_cols / filter

这三个入口最像 Polars，也是新手最先要掌握的执行上下文。

select: 只输出指定列

Python
df = qs.select(
    col("datetime"),
    col("code"),
    (col("close") / col("open") - 1).alias("ret"),
)

out = df.calc_data(data)

输入

datetime, code, open, close

输出

datetime, code, ret

with_cols: 在原数据上追加/覆盖列

Python
df = qs.with_cols(
    (col("close") / col("open") - 1).alias("ret"),
    col("close").shift(1).over("code").alias("prev_close"),
)

out = df.calc_data(data)

输入

原始 data 的所有列

输出

原始所有列 + ret + prev_close

filter: 过滤行

Python
df = (
    qs.with_cols((col("close") / col("open") - 1).alias("ret"))
    .filter(col("ret") > 0)
)

out = df.calc_data(data)
注意列顺序：如果你在同一个链路中先生成新列再过滤，确保过滤发生在生成之后。 上面写法先 with_cols 出 ret，再用 filter 过滤。

列选择与常量

col 是 Python 侧最重要的入口。它可以接收列名、多个表达式、Polars 表达式、Qust 表达式和常量包装。

写法 含义 输出宽度
col("close") 选择一列。 1 列
col("open", "close") 选择多列，后续可做跨列行算子或 plot 输入。 2 列
col.lit(1.0) 字面量常量。 1 列
col.null 空值表达式。 1 列
col.first() / col.last() 按输入列顺序选择第一列或最后一列。 1 列
col.nth(1) 按输入列下标选择第 2 列。 1 列

if_else 的常量陷阱

if_else 需要三列输入：条件列、True 分支、False 分支。数字常量必须写成 col.lit(...)。

Python
signal = col(
    col("close") > col("open"),
    col.lit(1.0),
    col.lit(-1.0),
).if_else().alias("signal")

out = qs.select(col("code"), signal).calc_data(data)
不要写：col(cond, -1, 1).if_else()。 这里的 -1 和 1 可能被解释成列索引，而不是你想要的常量。

算术、比较、逻辑

Qust 表达式支持常见 Python 运算符，写起来接近 Polars。

类别 运算符 例子
算术 + - * / // % ** (col("close") / col("open") - 1).alias("ret")
比较 > >= < <= == != col("volume") > 1000
逻辑 & | ^ ~ (col("ret") > 0) & (col("volume") > 1000)
空值相等 eq_missing / ne_missing col("a").eq_missing(col("b"))
Python
df = qs.select(
    col("code"),
    (col("close") / col("open") - 1).alias("ret"),
    ((col("close") > col("open")) & (col("volume") > 1000)).alias("strong_up"),
)

out = df.calc_data(data)

数学命名空间

ln、exp、log10、log 在 .math 命名空间，不在 .ta。

Python
df = qs.select(
    col("close").math.ln().alias("ln_close"),
    col("close").math.log(2.0).alias("log2_close"),
    col("close").math.sqrt().alias("sqrt_close"),
)

行方向计算

多列表达式可以做行方向聚合，比如每行取多列的均值、最大值、最小值。 这和 rolling 这种“跨行窗口”不同。

Python
df = qs.select(
    col("open", "close").mean(axis=1).alias("open_close_mean"),
    col("high", "low").max(axis=1).alias("high_low_max"),
    col("high", "low").min(axis=1).alias("high_low_min"),
)

out = df.calc_data(data)

输入

open, close, high, low 四列

输出

open_close_mean, high_low_max, high_low_min
判断方法：axis=1 是同一行的多列一起算；rolling(n) 是同一列的多行一起算。

rolling / expanding / over

窗口语义是本项目最容易写错的地方。下面按常见金融因子写法说明。

shift, diff, pct

Python
df = qs.select(
    col("code"),
    col("close"),
    col("close").shift(1).over("code").alias("prev_close"),
    col("close").diff(1).over("code").alias("delta_close"),
    col("close").pct(1).over("code").alias("ret_1"),
)

out = df.calc_data(data)
shift(n) 的 n 只能大于等于 0。 负数 shift 在流式场景里意味着看未来，当前实现会报错。

rolling: 固定长度窗口

Python
ma = col("close").mean().rolling(20, 1).over("code").alias("ma20")
vol = col("close").std().rolling(20, 2).over("code").alias("std20")

df = qs.select(col("datetime"), col("code"), ma, vol)
参数 含义 建议
window_size 窗口长度，可以是整数或 Params。 常用 5、10、20、60。
min_samples 窗口内非空样本不足时返回 null。 新手先设 1，生产因子按统计意义设置。

expanding: 从开始累计到当前

Python
df = qs.select(
    col("code"),
    col("volume").sum().expanding().over("code").alias("cum_volume"),
    col("close").mean().expanding().over("code").alias("mean_close_so_far"),
)

over: 分组上下文

over 可以包在窗口外层，让每个分组独立执行。常见分组是 code、datetime 或二者组合。

Python
# 每个 code 自己维护滚动窗口
ts_ma = col("close").mean().rolling(20, 1).over("code")

# 每个 datetime 做截面 rank
cs_rank = col("ret").batch_ta.rank().over("datetime")

# 多层分组: 每个行业、每个时刻单独排名
industry_rank = col("ret").batch_ta.rank().over("datetime", "industry")
不要机械替换 over。 时序算子一般 over("code")，截面算子一般 over("datetime")。 Alpha 公式里一个表达式树常常同时包含两种语义，必须逐个子表达式判断。

时序和截面

这里给出最常见的 Alpha 写法映射。把它当成速查表。

Alpha 伪代码 Qust 写法 语义
rank(x) x.batch_ta.rank().over("datetime") 同一时刻做截面排名。
ts_rank(x, n) x.rank().rolling(n).over("code") 每个标的看过去 n 行做时序排名。
delay(x, n) x.shift(n).expanding().over("code") 每个标的取过去第 n 个值。
delta(x, n) x - x.shift(n).expanding().over("code") 当前值减去过去值。
corr(x, y, n) col(x, y).corr().rolling(n).over("code") 每个标的滚动相关。

完整因子例子

Python
ret = (col("close") / col("close").shift(1).expanding().over("code") - 1).alias("ret")

alpha = (
    ret
    .batch_ta.rank()
    .over("datetime")
    .alias("alpha_rank")
)

df = qs.select(col("datetime"), col("code"), ret, alpha)
关于 batch_ta：rank、zscore、sort、lag、lead 这些是“批内/截面”语义， 通常需要放进 over("datetime") 或 batch("datetime") 一类上下文。

batch 与 snapshot_by

如果你的数据是分块来的，而且同一个标签可能在未来补到，普通 append 式状态计算可能不够。 这时要理解 batch 和 snapshot_by。

batch: 标签缓存和 update/retract

Python
expr = (
    col("alpha")
    .batch("datetime", keep_labels=128, start_calc_n=1)
    .with_cols(col("alpha").batch_ta.rank().alias("rank"))
)

df = qs.select(expr)
参数 说明
*by 标签列，至少一列。多列会组成联合标签键。
keep_labels 标签缓存上限，必须大于等于 1。
start_calc_n 每个标签累计到第 n 行才开始输出，必须大于等于 1。
need_active 激活后是否补齐激活前历史。更完整但更耗资源。

snapshot_by: 快照和回放

snapshot_by 偏向“结果一致性”。标签回流时，它会从快照基线开始回放历史，让状态算子收敛到一致结果。

Python
expr = (
    col("close")
    .mean()
    .rolling(3, 1)
    .over("code")
    .snapshot_by("datetime", keep_labels=256, start_calc_n=1)
    .alias("ma3")
)

df = qs.select(col("datetime"), col("code"), expr)
怎么选：如果目标是性能和 update/retract 管线，先看 batch。 如果目标是标签回流时保证状态一致，先看 snapshot_by。

交易行号和 get_by_index

交易类表达式经常先产生 entry/exit 行号，再回到原数据里取开仓价、平仓价、时间等 payload。 这个模式用 trade_row() 和 get_by_index(...) 完成。

最小例子

Python
trade_index = col("open_sig", "exit_sig").stra.trade_row()

df = qs.select(
    col("price").get_by_index(trade_index, keep=True)
)

out = df.calc_data(
    pl.DataFrame(
        {
            "price": [10, 20, 30, 40],
            "open_sig": [True, False, True, False],
            "exit_sig": [False, True, False, True],
        }
    )
)

输入

price:    10, 20, 30, 40
open_sig: true, false, true, false
exit_sig: false, true, false, true

输出

entry_index, exit_index, price-entry_index, price-exit_index
0,           1,          10,                20
2,           3,          30,                40

get_by_index 的硬规则

  • index 表达式的开头连续列必须是 UInt32，这些列被当作行号。
  • 后面非 UInt32 列是 payload，keep=True 时会保留。
  • 输出列名是 输入列名-index列名。
  • 普通上下文下，index 是整条输入的累计行号。
  • 外层 over(...) 下，index 是同一 group 内从 0 开始的组内行号。
  • index 子表达式自己不要再带 over(...)，应把 over(...) 放在外层。

保存中间结果与调试

复杂表达式链路里，最好先保存和打印中间结果，确认列宽、列名、类型和行数。

print(id)

print 会打印当前算子的真实输出，不改变表达式链路。

Python
df = qs.select(
    (col("close") / col("open") - 1)
    .alias("ret")
    .print("ret after calc")
)

out = df.calc_data(data)

save_data(key, pool) 和 pool.get_dataframe(key)

Python
pool = qs.DataPool("raw_pool")

df = qs.select(
    col("datetime", "code", "close").save_data("raw_close", pool),
    col("close").mean().rolling(2, 1).over("code").alias("ma2"),
)

_ = df.calc_data(data)
raw_close = pool.get_dataframe("raw_close")
print(raw_close)
适合场景：你怀疑某个上游表达式列数不对，或者 monitor 展示异常但不知道是不是数据包问题，先把中间 DataFrame 显式保存到 DataPool 里看。

读取与保存数据

本地 Python 和 Wasm 路径的 IO 语义不同，使用时要分清数据来自哪里。

本地 Python

Python
from qust import load_data, save_data

data = load_data("data/kline_data_all.parquet")

df = qs.select(
    col("datetime"),
    col("code"),
    col("close"),
)

out = df.calc_data(data)
save_data(out, "out.parquet")
函数 本地行为 返回
load_data(path) 调用 Polars 读取 parquet。 pl.DataFrame
save_data(data, file_name) 把 pl.DataFrame 写成 parquet。 包含文件名和大小的 dict。

Wasm / Pyodide

Wasm 下 load_data 不是直接读本地磁盘。普通名字要求数据已经在 Wasm 数据池里；HTTP/HTTPS 会走远程 parquet 加载。

Python in browser
# 数据已经在 Wasm 数据池中时
data = load_data("kline_data_all.parquet")

# 或远程 parquet
data = load_data("https://example.com/kline.parquet", name="kline")
文档口径提醒：当前 Python 内嵌说明倾向于“Wasm 普通名字必须在数据池中”。 如果写业务教程时希望直接从 web 根路径读裸文件名，需要先确认当前页面加载器是否真的注册了该数据。

流式状态和 snapshot

同一个执行计划对象会保存状态。对于 rolling、expanding、over、shift、UDF row 等算子，这个状态是功能的一部分。

Python
plan = qs.select(
    col("close").mean().rolling(3, 1).over("code").alias("ma3")
)

out1 = plan.calc_data(data.head(3))
snap = plan.snapshot()
out2 = plan.calc_data(data.tail(3))
out3 = snap.calc_data(data.tail(3))
对象 状态关系
plan 继续沿着第一次计算后的状态前进。
snap 从 snapshot 那一刻的状态分叉，之后和 plan 互不影响。
不要在不同实验之间复用同一个有状态 plan。 如果你要对同一份数据重新回测，重新构造计划最干净。

Monitor Dashboard 基础

Monitor 是浏览器 dashboard。它的前端只做 UI、canvas、输入事件、浮窗和 toolbar； 计算、参数重算、callback 查询都在 runtime 或表达式层完成。

最小折线图

下面是“slot-first”写法：先在 monitor 的 slot 上声明要显示什么图，session 启动时会把这些 plot 表达式编译进执行计划。

Python
monitor = (
    Monitor(background="white")
    .make_grid([["main"]])
    .select("main")
    .show(col("datetime", "close").monitor.line())
    .done()
)

df = qs.select(
    col("datetime"),
    col("code"),
    col("close"),
)

session = monitor.session(df, data).params().actions()
session.show_local(open_in_browser=True)
推荐主路径：monitor.session(df, data).params().actions()。 df.plot(data) 是快速绘图入口，适合调试和 wasm 工作台。

Grid 和 slot

make_grid 接收布局。相同 slot id 可以跨多个格子，表示一个图占多格。

Python
monitor = (
    Monitor(background="black")
    .make_grid([
        ["price", "price"],
        ["ret", "table"],
    ])
)

monitor = (
    monitor
    .select("price")
    .show(col("datetime", "close").monitor.line(), y_axis=0)
    .done()
    .select("ret")
    .show(col("datetime", "ret").monitor.bar())
    .done()
    .select("table")
    .show(col("code", "ret").monitor.table())
    .done()
)

表达式直接挂到 monitor

另一种写法是“expr-first”：monitor 只声明 grid，plot 表达式在 df 里通过 monitor(slot_id) 指定 slot。复杂链路里这通常更直观，因为计算列和图层写在同一条执行计划里。

Python
monitor = Monitor(background="black").make_grid([
    ["price", "price"],
    ["ret", "table"],
])

price_plot = col("datetime", "close").monitor("price").line()
ret_plot = col("datetime", "ret").monitor("ret").bar()

df = qs.with_cols(
    (col("close") / col("open") - 1).alias("ret"),
    price_plot,
    ret_plot,
)

monitor(slot_id, y_axis=0, y_ratio=1.0) 会把 plot 表达式绑定到指定 slot。

图表类型和输入列

图表输入列的顺序非常重要。第一列通常是 x 轴，后续列是 y 或专用字段。

Python 写法 最少列数 推荐输入 说明
.monitor.line() 2 x, y1, y2... 标准折线图，可多 y。
.monitor.bar() 2 x, y1, y2... 柱状图，可多 y。
.monitor.scatter(size=None) 2 x, y, payload... 散点图，xy range 视口，hover 取最近点。
.monitor.kline() 5 x, open, high, low, close K 线图。额外列可作为扩展信息。
.monitor("slot").fill(color, opacity) 2 或 3 x, y / x, col.null, y / x, y, col.null / x, y0, y1 2 列按折线填充到 0，跨 0 时按交点切分；3 列用 dtype=Null 派发到底部/顶部占位，否则填充 y0 到 y1。
.monitor.mark(shape, color, opacity, width) 2 x, y, flag? 标记图层。width 是相邻刻度间距占比，通常 0 到 1。
.monitor.histogram() 1 value 直方图。
.monitor.violin() 2 group, value 小提琴图。
.monitor.qq() 1 value QQ 图。
.monitor.area() 2 x, y1, y2... 面积图。
.monitor.step_line() 2 x, y 阶梯线。
.monitor.stacked_area() 2 x, y1, y2... 堆叠面积图。
.monitor.boxplot() 2 group, value 箱线图。
.monitor.heatmap() 3 x, y, value 热力图。
.monitor.table() 1 任意列 表格视图。

叠加图层例子

Python
df = qs.with_cols(
    col("datetime", "close").monitor("price").line(),
    col("datetime", "ma20").monitor("price", y_axis=0).line(),
    col("datetime", "upper", "lower").monitor("price").fill("#4a90e2", opacity=0.18),
    col("datetime", "signal_price", "signal").monitor("price").mark("triangle_up", color="red", width=0.5),

图表行为属于 plot 层能力。 不要在 monitor 前端按图类型硬写 hover、LOD、拖拽或降采样分支；这些应来自 plot behavior / render policy。

参数与动作

参数是 live 的。它们不是 Python 提前转成 int/float 的普通值，而是可以进入表达式树、由 monitor 参数面板驱动重算的对象。

创建参数

Python
window = pms(2, 120).title("window").value(20).step(1)
threshold = pms(0.0, 0.1).title("threshold").value(0.02).step(0.001)
side = pms(["long", "short", "both"], None).title("side").value("both")
参数类型 写法 注意
整数 pms(2, 120) start 和 end 都是 int。
浮点 pms(0.1, 5.0) 任一端为 float 即走浮点参数。
字符串枚举 pms(["a", "b"], None) 不支持 step。
表达式枚举 pms([expr1, expr2], "expr1") 第二参数可以是选中 Expr 名称或 None。

参数进入表达式

Python
window = pms(2, 120).title("ma window").value(20).step(1)

df = qs.select(
    col("datetime"),
    col("code"),
    col("close").mean().rolling(window, 1).over("code").alias("ma"),
)
不要提前写 int(window)。 这样会把 live 参数变成当前 Python 数字，monitor 参数面板之后就无法驱动这个表达式变化。

参数刷新事务

参数或动作触发重算时，monitor 前端按完整事务切图。

  1. 后端发 busy=true。
  2. 后端发 ResetRequested。
  3. 后端发送一批二进制 plot packets。
  4. 后端发 busy=false。
  5. 前端把 staged dashboard 原子切到前台。
为什么这样设计：避免参数更新时黑屏、图层只更新一半、mark/fill 大包闪烁。

slot callback

slot callback 是“某个图表区域自己的交互查询”。它和顶部 toolbar 的 monitor 全局参数/动作不是一回事。

x_slider 基础

Python
monitor = (
    Monitor(background="black")
    .make_grid([["a"]])
    .select("a")
    .show(col("datetime", "close").monitor.line())
    .source(col("datetime", "code", "close"), x="datetime")
    .on(x_slider())
    .done()
)

用户点击 slot 左上角按钮后，在图上拖选 x 区间，前端会弹出查询编辑器。查询结果统一返回一个 Monitor，显示在浮动结果窗口里。

直接运行 callback 代码

Python
callback_code = """
import qust as qs
from qust import col

result = qs.Monitor(background="white").make_grid([["ret"]])
result = (
    result
    .select("ret")
    .show(col("datetime", "close").monitor.line())
    .done()
)
result
"""

monitor = (
    Monitor(background="black")
    .make_grid([["a"]])
    .select("a")
    .show(col("datetime", "close").monitor.line())
    .source(col("datetime", "code", "close"), x="datetime")
    .on(x_slider(callback_code))
    .done()
)

attach_to_monitor

callback 可以附着到表达式树里的 monitor plot。绑定规则是结构规则，不靠变量名、标题或 slot 名。

表达式里 plot 数量 行为
0 个 报错，因为没有可绑定对象。
1 个 自动绑定。
多个 默认报错，除非提供明确 slot 约束。

scatter_select

scatter_select 和 x_slider 同属 callback，但 payload 语义不同。 x_slider 传 x 区间；scatter_select 传选中点的行值。

Python
monitor = (
    Monitor(background="black")
    .make_grid([["scatter"]])
    .select("scatter")
    .show(col("ret", "hold_period", "code", "entry_index", "exit_index").monitor.scatter(size=0.8))
    .source(col("datetime", "code", "ret", "hold_period", "entry_index", "exit_index"), x="datetime")
    .on(scatter_select())
    .done()
)
散点选择要带足 payload。 如果 callback 结果要回查交易，你的 scatter 输入或 source 里要带 code、entry_index、exit_index、收益、持仓周期等必要列。

Alpha / Alphalens

alphalen_analysis(monitor) 是链式表达式生成函数。它返回 Expr，不负责创建或替换 monitor。

推荐写法

Python
from qust import col, pms, Monitor

lead = pms(1, 20).title("lead").value(5)
quantiles = pms(2, 10).title("quantiles").value(5)

monitor = Monitor(background="white").make_grid([
    ["summary", "returns"],
    ["ic", "event"],
])

alpha_expr = (
    col("datetime", "code", "alpha", "ret")
    .alpha(lead=lead, quantiles=quantiles)
    .alphalen_analysis(monitor)
)

df = qs.select(alpha_expr)
session = monitor.session(df, data).params().actions()
session.show_local()

硬约束

  • monitor 必须由调用方传入，不要在 alphalen_analysis 内部偷偷创建。
  • 参数统一放在 .alpha(...)，并使用 live pms(...)。
  • lead、lag 这类参数必须直接吃 Params。
  • quantiles 不要在 Python 里 range(int(quantiles)) 展开，应走底层 list expr，让参数变化时列数自动跟着变。
  • 能复用的统计/聚合先落中间列，不要反复重算。

UDF

当内置表达式不够时，可以写 Python UDF。当前主要有两类：批处理 UDF 和逐行状态 UDF。

UdfBatch

UdfBatch 接收一个 Polars DataFrame，返回一个 Polars DataFrame。它适合一次处理一个批次。

Python
import polars as pl
import qust as qs
from qust import col, UdfBatch

class AddSpread(UdfBatch):
    def calc_batch(self, data: pl.DataFrame) -> pl.DataFrame:
        return data.with_columns(
            (pl.col("high") - pl.col("low")).alias("spread")
        ).select("spread")

df = qs.select(
    col("high", "low").udf.batch(AddSpread()).alias("spread")
)

UdfRow

UdfRow 有内部状态，适合逐行策略、在线累计、需要 update/calc/retract 的逻辑。

Python
from qust import UdfRow, DataType as dt

class RunningMean(UdfRow):
    def init_start(self):
        self.n = 0
        self.s = 0.0

    def output_schema(self, input_schema):
        return [("mean", dt.float64())]

    def update(self, value):
        if value is not None:
            self.n += 1
            self.s += float(value)

    def calc(self):
        return [None if self.n == 0 else self.s / self.n]
如果 UDF 要放进 rolling/dynamic/intra_day 这类需要回撤的上下文，必须认真实现 retract。 默认 retract 会报 NotImplemented。

设计思想：为什么是表达式

Qust 最重要的设计不是“把 Polars 包一层”，而是把你的计算意图保存成一棵表达式树。 用户写 col("close").mean().rolling(20).over("ticker") 时，表达式里同时保留了列、窗口、分组、参数和输出名。 这些信息留在表达式树里，后面才能统一做流式执行、参数重算、Wasm 运行和 Monitor 可视化。

Python 负责描述

Python API 尽量薄：负责把用户意图写成表达式，保留链式可读性，不在 Python 端提前做大量业务计算。

执行层负责计算

真正的数据计算、窗口状态、over 上下文、类型分发和二进制输出由底层执行层完成，行为更稳定。

Monitor 负责展示

表达式里说明“画什么”，图表执行层说明“怎么画”，前端负责 hover、zoom、select、slider 和结果窗口。

Wasm 负责可分享

同一套表达式可以搬到浏览器里跑，适合教学、演示、参数试验和轻量分享。

为什么不在 Python 里先算完

如果提前在 Python 算 保留成表达式的好处
参数变了要重新写一段流程。 pms(...) 可以进入表达式，参数面板改值后自动重算。
over、rolling、callback source 容易各自写一套逻辑。 上下文在表达式树里传播，分组和窗口语义更容易保持一致。
Notebook 能跑，Wasm 页面可能不能跑。 表达式可以序列化，Wasm 和本地 Python 复用同一套计划。
画图和计算混在一起，很难扩展交互。 plot expr 只管参数，plot exec 负责具体绘图，前端交互能力可以统一接入。

图表交互为什么也要统一

hover、zoom、select、query、slider 这些交互不应该散落在某个图类型里临时处理。 它们本质上都是“前端产生选择或查询请求，执行层根据上下文生成新结果”。统一之后，新增图表或 callback 时不需要推翻旧结构。

排错

这里按现象给出第一检查点。

现象 先检查 常见原因
参数拖动后图不变 控制面事务是否完整。 缺 busy=true、ResetRequested、packet 或 busy=false。
图层只显示一半 二进制 packet 和 render。 输入列数不满足图类型 arity，或 packet 解码失败。
scatter hover 不准 plot frontend behavior。 散点应走 nearest point，而不是 x-axis row。
callback 结果空 selection payload。 source 没配置，x 列不匹配，或 scatter payload 列不够。
Wasm 能跑最小例子但业务脚本失败 数据池、wheel、monitor assets、真实页面脚本。 只测了最小 callback，没覆盖真实 load_data 和 params/actions 流程。
calc_data 缺依赖 Arrow bridge。 没有安装 arro3-core 或 pyarrow。

真实回归 checklist

  • 不要只看 console 没报错，要看截图前后是否真的变化。
  • 参数回归要验证前图、参数变更、后图、截图不同。
  • slot callback 要验证按钮、真实拖选、高亮、编辑器、结果浮窗、结果 monitor 出图。
  • Wasm 大脚本要覆盖数据加载、策略表达式、monitor session、params、actions、slot source/on/x_slider。
